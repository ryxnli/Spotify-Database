# Spotify Database & Data Warehousing

## Overview

This academic MySQL project involved designing and querying a relational database for a Spotify-style music streaming platform. The database includes users, artists, albums, songs, playlists, payments, and streaming history.

## Project Objective

The goal of this project was to build a relational database, write SQL queries to analyze streaming and payment data, and create a basic star schema for reporting.

## Tools Used
- MySQL
- SQL
- Relational database design
- Data warehousing
- Star schema modeling

## What I Did
- Designed relational tables for users, artists, albums, songs, playlists, payments, and streaming history
- Wrote SQL queries using joins, filtering, grouping, subqueries, views, and aggregate functions
- Created a basic star schema using fact and dimension tables for streaming and payment reporting
- Organized the database structure to support analysis of user activity, songs, artists, and payments

- Relational database modeling
- Joins and aggregations
- Subqueries and views
- Data warehousing concepts
